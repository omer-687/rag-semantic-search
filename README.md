# RAG-Powered Semantic Search & QA — Pakistan History Chatbot

A three-stage Retrieval-Augmented Generation (RAG) system for question answering on Pakistan history. Combines Pinecone vector search with Tavily web search fallback, powered by LLaMA 3.3 70B via Groq API.

---

## Project Overview

This system answers natural language questions about Pakistan's history, culture, and politics using a hybrid retrieval approach. Rather than relying solely on a vector database or purely on web search, it intelligently routes each query through three stages to maximise answer quality.

---

## System Architecture

**Stage 1 — Query Classification**
A LLaMA 3.3 70B model first classifies whether the query is related to Pakistan. Off-topic queries are rejected immediately without consuming retrieval resources.

**Stage 2 — Pinecone Vector Search**
The query is embedded using `all-MiniLM-L6-v2` and searched against a Pinecone serverless index of chunked Pakistan history documents. A cosine similarity score is computed.

**Stage 3 — Adaptive Retrieval**
- If similarity score >= 0.65 — answer is generated from Pinecone RAG context
- If similarity score < 0.65 — system falls back to Tavily web search for fresh or obscure information

Final answers are generated by LLaMA 3.3 70B via Groq API with source attribution.

## Pipeline Diagram

```
User Query
    |
    v
[Stage 1] LLaMA Query Classifier
    |-- NOT Pakistan related --> Reject
    |
    v
[Stage 2] Pinecone Semantic Search
    |-- Similarity >= 0.65 --> RAG Answer from Vector DB
    |-- Similarity < 0.65  --> Tavily Web Search
    |
    v
[Stage 3] LLaMA 3.3 70B Answer Generation (Groq API)
    |
    v
Final Answer + Source Attribution
```

## Tech Stack

| Component | Tool |
|-----------|------|
| Vector Database | Pinecone (Serverless) |
| Embeddings | sentence-transformers (all-MiniLM-L6-v2) |
| Web Search Fallback | Tavily API |
| LLM | LLaMA 3.3 70B via Groq API |
| Text Chunking | LangChain RecursiveCharacterTextSplitter |

## Setup & Usage

### 1. Install Dependencies

```bash
pip install pinecone sentence-transformers tavily-python groq langchain-text-splitters
```

### 2. Set API Keys

Create a `.env` file or set environment variables:

```
PINECONE_API_KEY=your-pinecone-api-key
TAVILY_API_KEY=your-tavily-api-key
GROQ_API_KEY=your-groq-api-key
```

### 3. Run the Notebook

Open `OMER_BSAI23041_03.ipynb` in Jupyter or Google Colab and run all cells. The notebook will:
- Chunk and embed the Pakistan history dataset
- Upload vectors to Pinecone
- Run test queries through the full pipeline

## Key Design Decisions

**Similarity threshold at 0.65** — below this, the vector database doesn't have confident coverage, so Tavily web search provides fresher or more specific information rather than hallucinating from weak matches.

**Query classification first** — avoids wasting embedding and retrieval calls on off-topic queries entirely.

**Source attribution** — every answer includes whether it came from the vector database or web search, making the system transparent and debuggable.
